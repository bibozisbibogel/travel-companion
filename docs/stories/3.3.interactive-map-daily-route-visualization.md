# Story 3.3: Interactive Map with Daily Route Visualization

## Status
Completed

## Story
**As a** visual learner,
**I want** to see my daily activities and locations on an interactive map,
**so that** I can understand the geographic layout and optimize travel between locations.

## Acceptance Criteria
1. Interactive map integration (Google Maps) centered on destination
2. Day selector to filter map view by specific day or show entire trip
3. Accommodation markers with hotel name, rating, and price
4. Activity/restaurant pins with category-based color coding and icons
5. Route visualization showing travel paths between locations for each day
6. Clustering for multiple activities in close proximity
7. Pin click/hover reveals activity details: name, time, duration, brief description
8. Distance and estimated travel time between locations
9. Map controls: zoom, pan, fullscreen, map style toggle (standard, satellite)
10. Mobile-friendly touch interactions
11. Modern minimal map styling consistent with overall design
12. Legend showing activity category colors/icons
13. Current day highlight when viewing timeline

## Dev Notes

### Previous Story Dependencies
[Source: Story 3.2]
- Daily itinerary data structure provides location coordinates for markers
- Activity categories and color coding system must match Story 3.2
- Timeline and map components share state for day selection synchronization
- Activity details displayed in info windows mirror timeline card data

### Relevant Architecture Information
[Source: docs/architecture/tech-stack.md]
- **Frontend Framework**: Next.js 14 with React 18
- **Map Library**: Google Maps JavaScript API with React wrapper (@react-google-maps/api)
- **Marker Clustering**: @googlemaps/markerclusterer
- **Styling**: Tailwind CSS for map container and controls
- **State Management**: React Context or Zustand for map/timeline sync

### API Integration
[Source: docs/architecture/api-design.md]
- **Google Maps APIs**:
  - Maps JavaScript API for map rendering
  - Directions API for route visualization
  - Places API for location details (if needed)
  - Geocoding API for address to coordinates conversion
- **Trip Itinerary API**: `GET /api/trips/:tripId/itinerary` for location data

### Data Models
[Source: docs/architecture/data-models.md and Story 2.6]
- **Location Model**: latitude, longitude, address, place_id
- **ActivityMarker**: activity_id, name, category, location, time, duration, description
- **AccommodationMarker**: hotel_id, name, location, rating, price_per_night
- **RouteSegment**: origin, destination, distance, duration, mode (walk/drive/transit)

### Design System
- **Map Style**: Custom Google Maps style with neutral colors
- **Marker Colors** (matching Story 3.2):
  - Adventure: Blue (#3B82F6)
  - Cultural: Purple (#8B5CF6)
  - Relaxation: Green (#10B981)
  - Dining: Orange (#F59E0B)
  - Nightlife: Pink (#EC4899)
  - Shopping: Teal (#14B8A6)
  - Accommodation: Red (#EF4444)
- **Info Windows**: White background, subtle shadow, rounded corners
- **Route Colors**: Sequential color gradient or single accent color per day

### Google Maps Configuration
- **API Key Management**: Store in environment variables, restrict to domain
- **Map Options**:
  - Default Zoom: 13 (city level)
  - Min Zoom: 3, Max Zoom: 20
  - Gesture Handling: 'cooperative' for scroll behavior
  - Map Type Control: true (standard, satellite, terrain)
  - Fullscreen Control: true
  - Street View Control: false (keep minimal)

### Performance Considerations
- Lazy load Google Maps script to reduce initial bundle size
- Use marker clustering for 10+ markers in viewport
- Debounce route calculations when switching days
- Cache route data in component state or global store
- Consider server-side rendering limitations with maps

### Testing Standards
[Source: docs/architecture/coding-standards.md]
- **Test Location**: `packages/frontend/src/tests/components/`
- **Testing Framework**: Jest + React Testing Library
- **Map Testing**: Mock Google Maps API for unit tests
- **E2E Testing**: Playwright for full map interaction testing
- **Test Coverage**: Minimum 80% for map components
