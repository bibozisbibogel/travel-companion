# Story 2.2: Hotel Agent & Accommodation Search

## Status
Draft

## Story
**As a** travel planner,
**I want** the system to find suitable accommodation options,
**so that** I can choose hotels that match my preferences and budget.

## Acceptance Criteria
1. Hotel Agent implemented with Booking.com API integration
2. Hotel search with location, check-in/out dates, guest count, and budget filters
3. Hotel data model including amenities, ratings, location, and pricing
4. Integration with multiple hotel APIs (Expedia, Airbnb as fallbacks)
5. Hotel ranking algorithm considering price, location, and ratings
6. Cache mechanism for hotel search results

### Booking.com API Integration Requirements
[Source: architecture/external-apis.md]
**Booking.com API Configuration**:
- **Base URL**: `https://distribution-xml.booking.com/`
- **Authentication**: XML API with credentials
- **Rate Limits**: 100 requests/minute per property type
- **Primary Endpoint**: `POST /json/bookings.getHotelAvailabilityV2` for hotel availability search
- **Secondary Endpoint**: `POST /json/bookings.getHotelDescriptionPhotosV2` for hotel details and images
- **Integration Notes**: XML-based API requiring custom parsing, extensive property data available

### Data Models for Hotel Integration
[Source: architecture/data-models.md]
**HotelOption Entity Requirements**:
- **hotel_id**: UUID - Primary identifier for database storage
- **trip_id**: UUID - Foreign key relationship to Trip entity
- **name**: string - Hotel name for user display
- **price_per_night**: decimal - Nightly rate for budgeting and comparison
- **location**: Point - Geographic coordinates for proximity calculations
- **rating**: float - Hotel rating (1-5) for quality comparison
- **amenities**: JSON - Available amenities for user preference matching

### Database Schema Requirements
[Source: architecture/database-schema.md]
**Hotel Options Table Schema**:
```sql
CREATE TABLE hotel_options (
    hotel_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    trip_id UUID REFERENCES trips(trip_id) ON DELETE CASCADE,
    external_id VARCHAR(255), -- Booking.com/provider's ID
    name VARCHAR(255) NOT NULL,
    address TEXT,
    location POINT NOT NULL, -- PostGIS point for geographic queries
    price_per_night DECIMAL(10,2) NOT NULL,
    currency CHAR(3) DEFAULT 'USD',
    rating DECIMAL(2,1) CHECK (rating >= 1 AND rating <= 5),
    amenities JSONB DEFAULT '[]',
    photos JSONB DEFAULT '[]',
    booking_url TEXT,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
```

### Coding Standards Requirements
[Source: architecture/coding-standards.md]
**Python Naming Conventions**:
- **Classes**: PascalCase (`HotelAgent`, `BookingApiClient`)
- **Functions**: snake_case (`search_hotels`, `calculate_distance`)
- **API Endpoints**: kebab-case (`/api/v1/hotels/search-options`)
- **Database Tables**: snake_case (`hotel_options`, `external_id`)

**Critical Rules**:
- **Never log sensitive data**: No API keys, authentication tokens in logs - use structured logging with redaction
- **All external API calls must use circuit breakers**: Prevent cascade failures from Booking.com API outages
- **Database queries must use repository pattern**: Never direct ORM calls from agent classes
- **All async operations require timeout**: Prevent hanging requests from external services (30s default)
