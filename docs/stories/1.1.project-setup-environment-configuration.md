# Story 1.1: Project Setup & Environment Configuration

## Status
Approved

## Story
**As a** developer,
**I want** a properly configured development environment with all necessary dependencies,
**so that** I can begin building the travel companion application with consistent tooling.

## Acceptance Criteria
1. Python 3.11+ virtual environment created with UV package manager
2. FastAPI application structure established with basic routing
3. Next.js frontend project initialized with TypeScript and Tailwind CSS
4. Docker containers configured for both frontend and backend services
5. Environment variables configured for API keys and database connections
6. Development scripts created for running services locally

### Tech Stack Requirements
[Source: architecture/tech-stack.md]
- **Python**: 3.11+ for backend development with async support and LangGraph compatibility
- **FastAPI**: 0.104+ for API orchestration with high performance and async native support
- **Next.js**: 14.1+ for web application with React-based framework and excellent TypeScript support
- **TypeScript**: 5.3+ for frontend development with type safety
- **UV**: 0.1.15+ for Python dependency management (blazing fast, Rust-based)
- **Tailwind CSS**: 3.4+ for frontend styling with utility-first approach
- **Docker**: 24.0+ for development and deployment consistency
- **Pydantic**: 2.5+ for data models and validation with runtime type checking

### Project Structure Requirements
[Source: architecture/source-tree.md]
The project follows a monorepo structure with packages:
- **Backend API**: `packages/api/src/travel_companion/` with FastAPI app entry at `main.py`
- **Frontend Web**: `packages/web/src/app/` using Next.js App Router structure
- **Infrastructure**: `infrastructure/docker/` for containerization configs
- **Scripts**: `scripts/` directory for development automation
- **Root Configuration**: Root `package.json` for workspace, `docker-compose.yml` for quick start

### Coding Standards
[Source: architecture/coding-standards.md]
- **Languages & Runtimes**: Python 3.11+, TypeScript 5.3+, strict type checking enabled
- **Style & Linting**: Ruff for Python (Black-compatible), ESLint + Prettier for TypeScript
- **Naming Conventions**: 
  - Python Classes: PascalCase (`FlightAgent`, `TripPlanningWorkflow`)
  - Python Functions: snake_case (`search_flights`, `calculate_budget`)
  - TypeScript Interfaces: PascalCase with I prefix (`IFlightOption`, `ITripRequest`)
  - API Endpoints: kebab-case (`/api/v1/trips/search-flights`)

### Infrastructure Requirements
[Source: architecture/infrastructure-and-deployment.md]
- **Tool**: Docker Compose 2.20+ with container-first approach
- **Development Strategy**: Local Docker containers with hot reload
- **Pipeline Location**: `.github/workflows/` directory for CI/CD configuration
- **Environment Files**: `.env.example` for environment variable templates

### Testing Framework Setup
[Source: architecture/test-strategy-and-standards.md]
- **Python Testing**: pytest 7.4+ with async support
- **Frontend Testing**: Vitest 1.2+ for TypeScript (referenced in tech-stack.md)
- **Test Organization**: Tests adjacent to source code with `tests/` directories
- **Coverage Requirements**: Focus on business logic coverage
- **File Convention**: `test_*.py` files for Python, following pytest standards

### Security and Configuration
[Source: architecture/coding-standards.md]
- **Critical Rule**: Never log sensitive data - no API keys, passwords, or PII in logs
- **Environment Variables**: Use structured configuration with proper redaction
- **External Dependencies**: All external API calls must use circuit breakers and timeouts
