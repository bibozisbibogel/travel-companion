# Story 1.6: User Session & Authentication State Management

## Status
Complete

## Story
**As a** logged-in user,
**I want** my authentication state to persist across page refreshes and be accessible throughout the application,
**so that** I can access protected features without having to log in repeatedly.

## Acceptance Criteria
1. AuthContext provider manages global authentication state
2. useAuth hook provides access to current user and auth methods
3. Token persistence and automatic restoration on page load
4. getCurrentUser API endpoint fetches user profile after login
5. Logout functionality clears session and redirects to login
6. Protected routes redirect unauthenticated users to login
7. Header displays user profile when authenticated
8. Loading states during authentication checks

## Dev Notes

### Relevant Architecture Information
[Source: docs/architecture/tech-stack.md]
- **State Management**: React Context API for authentication state
- **Storage**: localStorage for JWT token persistence
- **API**: REST endpoints for user authentication

### API Integration
[Source: docs/architecture/api-design.md]
- **Login Endpoint**: `POST /api/v1/users/login` - Returns JWT token and user data
- **Register Endpoint**: `POST /api/v1/users/register` - Creates user and returns token
- **Current User Endpoint**: `GET /api/v1/users/me` - Returns current user profile (requires auth)
- **Token Format**: `Bearer {jwt_token}` in Authorization header

### Context Structure
```typescript
interface AuthContextType {
  user: User | null
  loading: boolean
  error: string | null
  login: (credentials: LoginFormData) => Promise<void>
  register: (data: RegisterFormData) => Promise<void>
  logout: () => void
  refreshUser: () => Promise<void>
  isAuthenticated: boolean
}
```

[Source: docs/architecture/coding-standards.md]

- **Testing Framework**: Jest + React Testing Library
- **Test Coverage**: Minimum 80% for auth context and hooks
