# Story 3.2: Day-by-Day Itinerary Timeline Visualization

## Status
Completed

## Story
**As a** user,
**I want** to see my complete trip organized by day with a clear timeline,
**so that** I can understand my daily schedule and flow of activities.

## Acceptance Criteria
1. Calendar/timeline view showing all trip days in sequence
2. Each day card displays: date, day of week, daily summary
3. Activities scheduled by time of day (morning, afternoon, evening, night)
4. Activity cards show: name, duration, estimated time, category icon, brief description
5. Meal recommendations integrated into timeline (breakfast, lunch, dinner)
6. Accommodation information displayed for each day (hotel name, location, check-in/out if applicable)
7. Daily budget breakdown showing estimated costs per day
8. Visual indicators for activity types using color coding or icons
9. Expand/collapse functionality for detailed vs. summary views
10. Modern minimal card-based design with clean typography
11. Smooth transitions and animations for expanding/collapsing content
12. Responsive layout adapting to screen size
13. Print-friendly view option

## Dev Notes

### Previous Story Dependencies
[Source: Story 3.1]
- Trip preferences data structure from Story 3.1 feeds into itinerary generation
- User's selected currency and budget constraints apply to daily budget display
- Activity type preferences influence activity recommendations shown in timeline

### Relevant Architecture Information
[Source: docs/architecture/tech-stack.md]
- **Frontend Framework**: Next.js 14 with React 18
- **UI Component Library**: shadcn/ui for modern minimal components
- **Styling**: Tailwind CSS with custom timeline design
- **Icons**: Lucide React or Heroicons for activity category icons
- **State Management**: React Context or Zustand for timeline state

### API Integration
[Source: docs/architecture/api-design.md]
- **Get Itinerary Endpoint**: `GET /api/trips/:tripId/itinerary` - Fetch complete itinerary
- **Itinerary Data Structure**: Array of daily schedules with activities, meals, accommodation
- **Response Format**: JSON with nested structure for days, time slots, activities

### Data Models
[Source: docs/architecture/data-models.md and Story 2.6]
- **DailyItinerary Model**: date, day_of_week, activities[], meals[], accommodation, daily_cost
- **ItineraryItem Model**: name, type, time_slot, duration, location, description, price, category
- **Meal Model**: restaurant_name, cuisine_type, meal_type, time, price_range
- **Accommodation Model**: hotel_name, location, check_in, check_out, price_per_night, rating

### Design System
- **Color Coding for Activities**:
  - Adventure: Blue (#3B82F6)
  - Cultural: Purple (#8B5CF6)
  - Relaxation: Green (#10B981)
  - Dining: Orange (#F59E0B)
  - Nightlife: Pink (#EC4899)
  - Shopping: Teal (#14B8A6)
- **Typography**: Clean sans-serif, clear size hierarchy (H2 for days, H3 for time slots)
- **Card Design**: Subtle shadows, rounded corners, hover states
- **Animations**: 200-300ms ease-in-out transitions

### Testing Standards
[Source: docs/architecture/coding-standards.md]
- **Test Location**: `packages/frontend/src/tests/components/`
- **Testing Framework**: Jest + React Testing Library
- **Test Coverage**: Minimum 80% for timeline components
- **Visual Regression**: Consider Percy or Chromatic for UI testing
