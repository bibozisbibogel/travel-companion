# Story 3.5: User Trip List Dashboard

## Status
Completed

## Story
**As a** returning user,
**I want** to see a list of all my trips after logging in,
**so that** I can quickly access my past and planned trips.

## Acceptance Criteria
1. Display list of all user trips after successful login
2. Each trip card shows: trip name, destination, dates, status, and thumbnail/preview
3. Trips sorted by creation date (most recent first)
4. Pagination support for users with many trips (20 trips per page)
5. Filter trips by status (draft, planning, confirmed, completed, cancelled)
6. Search trips by destination or trip name
7. Click on trip card navigates to trip detail page
8. Empty state message when user has no trips with call-to-action to create first trip
9. Loading state while fetching trips from API
10. Error handling with retry functionality if API call fails
11. Modern minimal card-based design consistent with app aesthetic
12. Mobile-responsive grid layout (1 column mobile, 2-3 columns tablet/desktop)

## Dev Notes

### Backend API Already Implemented
[Source: packages/api/src/travel_companion/api/v1/trips.py#L149-L215]
- **Endpoint**: `GET /api/v1/trips`
- **Authentication**: Required via JWT token (Authorization header)
- **Query Parameters**:
  - `page` (int, default: 1) - Page number
  - `per_page` (int, default: 20, max: 100) - Items per page
- **Response Model**: `PaginatedResponse[list[TripResponse]]`
- **Response Structure**:
  ```json
  {
    "data": [
      {
        "trip_id": "uuid",
        "user_id": "uuid",
        "name": "Trip to Tokyo",
        "description": "AI-generated travel plan for Tokyo",
        "destination": {...},
        "start_date": "2024-06-01",
        "end_date": "2024-06-07",
        "total_budget": 3000.00,
        "status": "draft",
        "itinerary_data": {...},
        "created_at": "2024-03-15T10:30:00Z",
        "updated_at": "2024-03-15T10:30:00Z"
      }
    ],
    "pagination": {
      "page": 1,
      "per_page": 20,
      "total_items": 45,
      "total_pages": 3,
      "has_next": true,
      "has_prev": false
    },
    "message": "Trips retrieved successfully"
  }
  ```

### Data Models
[Source: docs/architecture/data-models.md]
- **Trip Model**:
  - trip_id: UUID - Primary identifier
  - user_id: UUID - Foreign key to User
  - name: string - Trip name
  - destination: string - Primary travel destination
  - start_date: date - Trip start date
  - end_date: date - Trip end date
  - budget: decimal - Total trip budget
  - status: enum - (draft, planning, confirmed, completed, cancelled)
  - itinerary_data: JSON - Complete itinerary with all bookings

### Database Schema
[Source: docs/architecture/database-schema.md]
- **trips table** has Row Level Security (RLS) enabled
- Users can only access their own trips via policy: `user_id = auth.uid()`
- Status enum values: 'draft', 'planning', 'confirmed', 'completed', 'cancelled'
- Trips are sorted by `created_at` timestamp by default
